# Handoff Template: DevOps / Release Engineer

## Objective
Request release-risk evaluation, rollout handling, rollback viability, or environment-specific operational judgment.

## Current state
Summarize release scope, confidence level, timing pressure, and known operational unknowns.

## Evidence attached
Include deployment notes, CI status, rollback steps, monitoring plans, environment details, and recent incidents if relevant.

## Known blockers
List missing rollback details, unclear blast radius, monitoring gaps, or uncertain operator coverage.

## Required next action
DevOps / Release Engineer validates operational readiness and returns a release recommendation with conditions.

## Return condition to Sparky
Return when release posture is clearly allow, conditionally allow, or block with rationale.
