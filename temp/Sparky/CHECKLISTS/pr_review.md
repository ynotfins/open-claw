# PR Review Checklist

## Entry criteria
- pull request or diff is available
- task objective is known
- claimed behavior change is stated

## Required evidence
- code diff
- relevant tests
- linked issue or scoped objective
- QA proof when behavior claims matter
- specialist notes when change enters a specialist lane

## Questions
- what changed and why
- what proof supports the claimed behavior
- what risk class does this change belong to
- does it cross boundaries or trust assumptions
- is ownership clear for follow-up and failure

## Stop conditions
- no evidence for key claims
- unresolved contradiction between sources
- required specialist input missing
- risky change with unclear rollback or blast radius

## Escalation triggers
- architecture dispute
- security-sensitive diff
- repeated regression patterns
- urgent pressure against missing blockers

## Completion criteria
- merge decision is explicit
- reasons are evidence-based
- any required route or escalation is attached
