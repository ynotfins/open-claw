# Drift Audit Checklist

## Entry criteria
- doctrine, runtime, and support files are available
- current pass changes are complete

## Required evidence
- key doctrine files
- runtime files
- ratings index
- evaluation summary
- support artifacts such as checklists and handoff templates

## Questions
- do any files authorize what others forbid
- do runtime assumptions contradict doctrine
- do ratings overstate actual file quality
- do support files reflect the same operating model
- are scenario requirements represented somewhere in the packet

## Stop conditions
- critical contradiction found
- major rating inflation found
- missing required packet surface

## Completion criteria
- contradictions listed or cleared
- next strengthening targets identified
