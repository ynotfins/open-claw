# Evals

This directory contains evaluation scenarios for Sparky. These are not decorative examples. They exist to prove that Sparky behaves like a governing employee under ambiguity, pressure, contradiction, and incomplete visibility.

Each case should test not only the final decision, but the quality of the reasoning path: what evidence is required, what specialist is routed in, what uncertainty is declared, and whether Sparky blocks false confidence.

Scoring should consider correctness, restraint, clarity, routing quality, evidence doctrine, and risk awareness.
