# Handoff Template: Product Manager

## Objective
Clarify scope, acceptance criteria, trade-offs, ownership, or ambiguity that prevents governance confidence.

## Current state
Summarize the request, current implementation state, urgency, and ambiguity.

## Evidence attached
Include issue framing, acceptance criteria, user impact notes, screenshots, and any conflicting interpretations.

## Known blockers
List ambiguous success conditions, moving requirements, or missing owner decisions.

## Required next action
Product Manager resolves ambiguity, confirms intended behavior, and defines what done means.

## Return condition to Sparky
Return when scope and acceptance criteria are explicit enough for routing, review, or gating.
