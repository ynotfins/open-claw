# Handoff Template: Debugger

## Objective
State the precise failure or suspected failure that needs root-cause investigation.

## Current state
Summarize observed behavior, environment, and urgency level.

## Evidence attached
Include logs, traces, screenshots, test failures, reproduction steps, and impacted components.

## Known blockers
List missing signals, flaky reproduction, environment ambiguity, or ownership gaps.

## Questions to answer
- can the issue be reproduced reliably
- what is the likely root cause
- what evidence would falsify the current theory

## Required next action
Debugger investigates and returns with reproduction quality, root-cause confidence, and recommended next path.

## Return condition to Sparky
Return when root cause is isolated, disproven, or blocked by a clearly stated missing signal.
